# Impeccable Context

本文件是项目的长期设计规范，供前端设计类任务直接读取和执行。

- 它定义这个产品的视觉系统、组件规格和文案规则。
- 它不是 PRD、技术设计或实现 tracker 的替代品。
- 产品定位或品牌方向变化时需同步更新。
- 新的视觉决策必须在这里沉淀为可执行规则，不要依赖会话截图长期存在。

更新日期：2026-04-07

---

## 1. 产品定位

AnchorDesk 是私有资料库知识工作台。用户围绕明确主题（项目、客户、研究方向）组织资料、问答、沉淀结论。

核心气质：**可信、克制、安静**。它是工具，不是玩具。

视觉北极星：**The Digital Curator**（冷静、留白、editorial intelligence）。

## 2. 色彩系统

所有颜色通过 CSS 变量定义在 `apps/web/app/globals.css`（以当前实现为准）：

| 语义 | 变量 | 值 |
|------|------|------|
| 页面背景 | `--bg` | `#f7f9fb` |
| 顶部渐变高亮 | `--bg-elevated` | `#fbfcfd` |
| 低层表面 | `--surface-low` | `#f2f4f6` |
| 默认表面 | `--surface` | `#eceef0` |
| 高层表面 | `--surface-high` | `#e6e8ea` |
| 顶层表面 | `--surface-lowest` | `#ffffff` |
| 主文本 | `--text` | `#191c1e` |
| 次级文本 | `--secondary` | `#565e74` |
| 弱文本 | `--muted` | `#6d7488` |
| 次级弱文本 | `--muted-strong` | `#565e74` |
| AI/引用强调 | `--accent-beige` | `#f0e0cb` |
| Ghost border | `--outline-variant` | `#c6c6cd` |
| 选中/强调底 | `--secondary-fixed` | `#dae2fd` |
| 主操作 | `--primary` | `#000000` |
| 主操作反白 | `--primary-contrast` | `#f8f9fc` |
| 交互强调 | `--accent` | `#7a6d5b` |

规则：

- 默认不用实体分割线做区块分隔；边界优先用 surface 叠层与留白解决。
- 只有在白底叠白底、或可访问性需要时，才使用 ghost border（`--outline-variant` 的低不透明度）。
- 导航选中使用 anchor line（细竖线）表达，而不是整块卡片式高亮。
- 标题使用 headline sans（`font-headline`），不再把 serif 作为默认 heading；serif 只允许出现在少量品牌字标场景。

## 3. 字体与排版

字体变量在 `globals.css` 的 `@theme inline` 中声明（以当前实现为准）：

```css
--font-sans: "Inter", "PingFang SC", "Hiragino Sans GB", "Source Han Sans SC", "Microsoft YaHei", sans-serif;
--font-headline: "Manrope", "PingFang SC", "Hiragino Sans GB", "Source Han Sans SC", "Microsoft YaHei", sans-serif;
--font-serif: "Iowan Old Style", "Baskerville", "Songti SC", "Noto Serif SC", serif; /* 仅品牌字标等少量场景允许 */
```

排版规则：

- 全局 body 使用 `--font-sans`。
- 全局 `h1/h2/h3` 使用 `--font-headline`，并带较紧的 tracking（见 `globals.css`）。
- 页头统一使用 [`apps/web/components/shared/editorial-page-header.tsx`](apps/web/components/shared/editorial-page-header.tsx)：
  - eyebrow：`text-[11px] font-semibold uppercase tracking-[0.18em]`
  - title：`text-[2.6rem] font-extrabold tracking-[-0.05em]`
  - description：`text-[15px] leading-7`
- 常规 UI 文本优先保持高信息密度：`text-[13px]`~`text-[14px]` 为主。

文案规则：

- 短文案不加句号。
- 不写"欢迎语、功能复述、路径解释"等废话。
- 标题、按钮已经表达清楚的，不再补一行重复。
- 删掉后不影响操作和理解的文案，就不应出现。

## 4. 深度与阴影

阴影变量以 `globals.css` 为准：

```css
--shadow-soft: 0 8px 18px rgba(18, 24, 32, 0.05);
--shadow-card: 0 14px 30px rgba(18, 24, 32, 0.08);
```

规则：

- 深度优先靠 surface 叠层（tonal layering）；阴影只做极轻补充。
- panel/popover/dialog 的边界优先使用 ghost border，而不是实边框 + 重阴影。

## 5. 圆角体系

圆角以 `apps/web/lib/ui.ts` 为准（避免散写魔法数）：

| 用途 | 圆角 |
|------|------|
| 按钮/输入框 | `rounded-xl`（12px） |
| popover/menu/dialog | `rounded-2xl`（16px） |
| sectionPanel | `rounded-[18px]` |
| subpanel | `rounded-[16px]` |
| subcard | `rounded-[14px]` |
| workspace tile（列表卡） | `rounded-[20px]`（仅此类 tile） |
| pill/chip | `rounded-full` |

规则：

- 以 12/14/16/18 为主，避免 24px+ 的“厚卡片”观感。
- popover/menu/dialog 统一 16px，且搭配 ghost border + glass surface。

---

## 6. 间距节奏

默认页面容器使用 `ui.page` / `ui.pageNarrow`（`apps/web/lib/ui.ts`）：

- 页面纵向节奏：`gap-4`
- 页面内边距：`px-3 py-5`，桌面 `md:px-5 md:py-6`
- 动作区：`ui.actions`（`gap-1.5`）
- 工具条：`ui.toolbar`（filled surface + 轻圆角）

避免 `gap-8` 以上的大间距。内容靠上组织，操作贴近内容。

## 7. 组件规格

### 7.1 Buttons

统一使用 `buttonStyles()`（`apps/web/lib/ui.ts`）。

variant 契约（以当前实现 className 为准）：

- `primary`: `bg-app-primary text-app-primary-contrast`，hover 进入更深的 charcoal（当前实现为 `hover:bg-[#25211c]`）
- `secondary`: integrated fill（当前实现为 `bg-app-surface-lowest text-app-secondary hover:bg-app-surface`）
- `ghost`: 透明底，hover surface shift（当前实现为 `hover:bg-app-surface-high/65`）
- `danger`: `bg-red-600 text-white`
- `dangerGhost`: `text-red-600 hover:bg-red-50`

size 契约：

- `md`: `min-h-10 px-3.5 text-[14px]`
- `sm`: `min-h-8 px-3 text-[13px]`
- `xs`: `min-h-7 px-2.5 text-[12px]`
- icon：`size-10 / size-8 / size-7`

规则：弹出面板内只用 `xs` 或 `sm`；同一行避免多个同级实底按钮竞争焦点。

### 7.2 Fields

输入体系统一使用 filled field（`inputStyles()` / `textareaStyles()` / `selectStyles()`）。

规则（以当前实现为准）：

- 默认：`bg-app-surface-low border-transparent`
- focus：ghost border（`outline-variant` 低不透明度）+ `secondary-fixed` ring + offset

### 7.3 Surfaces

统一使用 `ui.panel` / `ui.panelLarge` / `ui.sectionPanel` / `ui.subpanel` / `ui.subcard`（`apps/web/lib/ui.ts`）。

规则：

- panel 允许轻玻璃（`surface-lowest` 的低不透明度 + `backdrop-blur`），但默认不使用“实边框卡片”。
- section/sub 允许使用 outline-variant 的低不透明度作为 ghost border。

### 7.4 Popover / Menu / Dialog

统一使用 `ui.popover` / `ui.menu` / `ui.dialog`：ghost border + glass surface + blur。

规则：popover/menu 内默认不再嵌套子卡片；结构靠排版与留白组织，不靠分割线堆叠。

### 7.5 Navigation

- 侧栏/列表选中使用 anchor line（见 `navItemStyles()`）。
- “新建会话”按钮选中态同样使用 anchor line（见 `createConversationNavButtonStyles()`）。

## 8. 页面级布局

页面 family 通过共享 shell 保持一致：

- workspace family：`WorkspaceShellFrame`（侧栏 + sticky top bar + content scroll）。
- management family：`SettingsShell`（sticky frosted top bar + 侧栏导航）。
- auth family：`AuthShell`（双栏布局，右侧聚焦表单）。
- public family：`PublicPageShell`（sticky frosted header + 窄内容带）。

## 9. 文本选择策略

默认用 `apps/web/lib/ui.ts` 的 `textSelectionStyles` 控制选择行为：

- UI chrome（导航、按钮、工具条容器等）优先用 `textSelectionStyles.chrome`（`select-none`）。
- 承载知识内容的区域（问答正文、引用摘录、文档正文、可复制字段）显式用 `textSelectionStyles.content`（`select-text`）。
- 不做全局强制开关，按组件语义标注，避免误触复制影响交互。

目标：

- 弱化“网页 DOM 文本到处可选”的感觉
- 提升界面的安定感、完整度和工具质感
- 让“可复制”成为内容语义，而不是浏览器默认噪音

---

## 10. 不做的事

- 不做霓虹、赛博、渐变发光、深色面板。
- 不做消费化运营元素（积分卡、勋章、活动位）。
- 不做大气泡、大头像聊天界面。
- 不做极端稀疏的展示型布局。
- 不在弹出面板中嵌套带圆角和阴影的子卡片。
- 不写"解释面板用途"的冗余帮助文案。
- 不在同一行放多个同级别实底按钮竞争视觉焦点。
- 不使用 TailwindCSS 默认蓝色系颜色。

---

## 11. 设计决策原则

1. **可测量**：本文件的规则应具体到 class name 或数值，不写"适当留白""合理圆角"。
2. **单一来源**：颜色、间距、圆角的规格以本文件和 `lib/ui.ts` + `globals.css` 为准。
3. **减法优先**：字段、说明、装饰、交互先压到最小集合。
4. **一屏一主任务**：每个页面一个清晰主目标，不做多个同级 CTA 竞争。
5. **规则可沉淀**：新的视觉偏好必须提炼为这里的明确规则，不停留在"像某产品"。
